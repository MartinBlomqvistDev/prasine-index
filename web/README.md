# Prasine Index — Web Frontend

Next.js 15 (App Router) frontend for the Prasine Index greenwashing monitoring pipeline.

## What it does

- **Public report viewer** (`/`) — renders canonical greenwashing assessment reports for assessed companies. Multi-claim layout: aggregate company verdict, claim summary table, detailed assessment of the highest-scoring claim with full evidence chain.
- **Dashboard** (`/dashboard`) — internal view for pipeline operators. Lists all assessed companies with scores and verdicts.
- **Request form** (`/dashboard/request`) — intake form for new company assessment requests (Formspree).
- **Auth** (`/sign-in`, `/sign-up`) — Clerk-based authentication gating dashboard access.

## Stack

| Layer | Choice |
| ----- | ------ |
| Framework | Next.js 15 App Router |
| Auth | Clerk |
| Styling | CSS modules (no Tailwind) |
| Forms | Formspree |
| Hosting | Vercel |

## Development

```bash
cd web
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Build

```bash
npm run build
```

Build output goes to `.next/`. Deploy via Vercel (auto-deploys on push to `main`).

## Report format

The main page (`app/page.tsx`) renders hardcoded canonical reports. Reports are generated by the Python pipeline (`scripts/run_assessment.py`) and saved to `docs/reports/<slug>.md`. The web layer reads them statically — no database connection, no API calls, no runtime dependencies beyond Next.js.

To update a displayed report after a pipeline re-run: copy the updated evidence, scores, and reasoning from the canonical `.md` file into the corresponding section of `app/page.tsx`.
